How to Use Dev Starter V1

Step-by-step guide for installation, configuration, and daily usage

1. Installation

Option A — Clone from GitHub

# Clone the repository
git clone https://github.com/clickzika/dev-starter.git

# Copy everything to Claude Code's config directory
cp -r dev-starter/* ~/.claude/

# Windows (PowerShell):
# Copy-Item -Recurse dev-starter\* $env:USERPROFILE\.claude\

Option B — Import from Backup

# If you have a dev-starter-v1.zip backup:
claude
> /import ~/Desktop/dev-starter-v1.zip

2. First-Time Setup

1
Run Setup Script
Checks prerequisites (git, gh, node, docker) and configures your environment.
bash ~/.claude/setup.sh
2
Configure Secrets
Copy the example file and fill in your tokens. Never commit .env.
cp ~/.claude/.env.example ~/.claude/.env
VariableWhere to GetRequired?
GITHUB_USERNAMEYour GitHub profileRequired
GITHUB_TOKENgithub.com/settings/tokens (scope: repo)Or use gh auth login
NOTION_API_KEYnotion.so/my-integrationsOptional
PROJECTS_ROOTWhere project folders are createdDefault: ~/Projects
3
Fill Your Profile
Edit ~/.claude/USER.md with your skill levels. Agents auto-calibrate based on this.
Your LevelHow Agents Respond
BeginnerFull explanations, define all terms, warnings, step-by-step
IntermediateBrief explanation + code, skip basics, highlight non-obvious
AdvancedCode + trade-offs only, no hand-holding, edge cases only
ExpertDense output, assume full context, non-trivial only
4
Configure Team (Optional)
Edit ~/.claude/TEAM.md if you work with others. Defines ownership, PR rules, decision authority.

3. Starting a New Project

# Open Claude Code
claude

# Launch the menu
> Read ~/.claude/dev-menu.md and help me get started

# Select option 1 (New Project)
# The system walks you through 5 gates automatically
What Happens Automatically Gate 1 asks you 26 questions about your project. From your answers, agents create: CLAUDE.md, BRD, SRS, DB design, API reference, security design, infrastructure guide, test strategy, UI prototype with component library, and a project plan.

For Agents — Registering New Documents

Rule: Every new HTML doc in docs/ must also be added to docs/index.html Add your document to the DOCUMENTS array in the <script> section, under the appropriate section. 
// In docs/index.html — find the DOCUMENTS array and add:
{
  section: "Technical Design",
  docs: [
    { name: "Your Document Title", file: "your-doc.html", color: "var(--cat-other)" },
  ]
}

7. Backup & Transfer

Export (backup)

# In Claude Code:
> /export                    # saves to Desktop
> /export C:\backup          # saves to specific folder

Import (restore on new machine)

# In Claude Code:
> /import                                  # from Desktop
> /import C:\backup\dev-starter-v1.zip     # from specific file
Secrets are NOT included in exports After importing on a new machine, you must create ~/.claude/.env again with your tokens. Run gh auth login to authenticate GitHub CLI.

8. Keeping CLAUDE.md Fresh

As your project evolves, CLAUDE.md may become stale. Use the context command to refresh it:

# In Claude Code:
> /context

This scans the current codebase and proposes updates to CLAUDE.md to match the actual state of the project.

9. Troubleshooting

ProblemSolution
Agent doesn't know the project contextCheck that CLAUDE.md exists at project root. Run /context to refresh.
GitHub integration not workingRun gh auth status. If not authenticated, run gh auth login.
Notion integration not workingCheck ~/.claude/.env has NOTION_API_KEY. Verify integration is connected to your workspace.
Agent explanations too basic / too advancedUpdate your skill levels in ~/.claude/USER.md.
Missing workflow filesCheck ~/.claude/sdlc/ directory. Re-import if needed: /import.
Export/Import failsMake sure you have write permission to the target directory.
