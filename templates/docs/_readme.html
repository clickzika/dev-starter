How to Use Dev Starter V1

Step-by-step guide for installation, configuration, and daily usage

1. Installation

Option A — One Command (recommended)

git clone https://github.com/clickzika/dev-starter.git && bash dev-starter/install.sh

This single command will: clone the repo, backup existing ~/.claude/ files, copy everything, and run the setup wizard.

Option B — Manual Install

# Clone
git clone https://github.com/clickzika/dev-starter.git

# Copy to Claude Code's config directory
cp -r dev-starter/* ~/.claude/

# Run setup
bash ~/.claude/setup.sh

4. The 5-Gate Build Process

When you type /new, the system walks you through 5 gates:

GateWhat HappensApproval
Gate 0GitHub repo, Notion board, branch strategy, templatesAuto (no approval needed)
Gate 126 questions → CLAUDE.md, BRD, SRS"approve"
Gate 2DB schema, API, security, infra, test strategy, UI prototype"approve"
Gate 3Task breakdown, GitHub Issues + milestones, Notion tasks, scaffold"approve"
Gate 4Feature-by-feature: branch → code → PR → review → merge"approve" per feature
Gate 5Testing, security audit, performance review"approve"

5. Release Flow

After Gate 5, use /release for the full deployment pipeline:

develop ──→ Local Test ──→ uat ──→ User Test ──→ main ──→ Production
              (SIT)                  (UAT)                 (PROD)
GateUser TypesWhat Happens
Gate 1"DEV approved"Pre-release checklist passed → proceed to SIT
Gate 2"SIT approved"Automated tests passed (local Docker) → proceed to UAT
Gate 3"UAT approved"User tested manually → proceed to Production
Gate 4"DEPLOY v[X.Y.Z]"Deploy to production

6. Usage Examples

Example 1 — Build a new web app from scratch

claude

# Claude reads CLAUDE.md, asks what to change

# After merge: backports to uat + develop automatically

Example 4 — Plan next sprint

claude

9. Deploying Project Docs

GitHub Pages (public repos)

# In your project repo settings:
# Settings → Pages → Source: Deploy from branch → /docs → Save
# Your docs will be at: https://username.github.io/project-name/

Cloudflare Pages (private repos — free)

  1. Go to dash.cloudflare.com → Workers & Pages → Create → Pages
  2. Connect to Git → select your repo
  3. Production branch: develop (or main), Build command: (empty), Output: docs
  4. Save and Deploy → https://project-name.pages.dev

10. Backup & Transfer

# Export (backup)
> /export                    # saves to Desktop

# Import (restore on new machine)
> /import                    # from Desktop
Secrets are NOT included in exports After importing on a new machine, run bash ~/.claude/setup.sh to configure credentials.

11. Troubleshooting

ProblemSolution
Agent doesn't know project contextCheck CLAUDE.md exists. Run /context to refresh.
GitHub integration not workingRun gh auth status. If not authenticated: gh auth login.
Notion integration not workingCheck ~/.claude/.env has NOTION_API_KEY.
Agent explanations too basic/advancedUpdate skill levels in ~/.claude/USER.md or re-run setup.sh.
Claude keeps asking permissionRun setup.sh again — it configures auto-permissions.
Rate limit interrupted workAuto-resume via checkpoint. Or type "ทำต่อ" when Claude resets.
Missing workflow filesCheck ~/.claude/sdlc/. Re-import: /import.
