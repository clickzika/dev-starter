Claude Code — Full Team Setup

15-Agent World-Class Software Development Team
15 Agents 10 Commands 5-Gate Workflow

A complete agent team for Claude Code covering every role in software development. Agents are self-improving, context-aware, and follow a structured approval workflow that connects automatically to Notion and GitHub.

Quick Start

  1. One-time setup: run ~/.claude/setup.sh and fill in ~/.claude/.env
  2. Edit your profile: ~/.claude/templates/USER.md
  3. Open Claude Code: claude
  4. Say: /build ProjectName description of what to build
  5. Everything is auto-created — folder, GitHub repo, Notion board, all docs

The 3 Ways to Use the Team

Way 1 — Single Agent

Call one specialist directly for a specific task.

@frontend build the login screen
@dba optimize this slow query
@security review this auth code

Way 2 — Orchestrator Command

One prompt, the whole team works through it in sequence.

/build Company internal website
/feature Add dark mode toggle
/fix Login fails after 30min idle

Way 3 — Full Project Workflow

New projects with BRD, SRS, prototype, Notion tasks, and GitHub — use the 5-Gate Workflow.

The 5-Gate Approval Workflow

Nothing happens without your explicit approval at each gate.

GATE 1 Documents
@ba → BRD  |  @ba + @techlead → SRS  |  @dba → DB Design  |  @backend → API Ref  |  @security → Security Design  |  @devops → Infra  |  @qa → Test Strategy  |  @uxui → Prototype  |  @pm → Project Plan
You say: "Approved"
↓
GATE 2 Task Breakdown
@pm → Epic → Feature → Task list. Each task: title, AC, effort, role, dependencies
You say: "Task list approved"
↓
GATE 3 Notion + GitHub Setup
Notion: database + all tasks  |  GitHub: repo + issues + milestones  |  Link: Notion ↔ GitHub
You say: "Confirmed. Start."
↓
GATE 4 Per-Task Development
Agent states task + branch → You say "go" → Agent builds + opens PR → You approve PR
You say: "go" per task
↓
GATE 5 Completion
Agent merges PR → Updates Notion → Done → Closes GitHub issue → Asks for next task
Automatic after PR approval

Skill Calibration

Fill in ~/.claude/templates/USER.md with your skill levels. Every agent adapts:

Your LevelHow Agents Respond
BeginnerFull explanations, define all terms, warn about common mistakes
IntermediateCode + brief explanation, skip basics, highlight non-obvious parts
AdvancedDense output, trade-offs only, skip all fundamentals
ExpertJust the code and the edge cases — no hand-holding

Notion + GitHub Integration

Tokens are stored globally in ~/.claude/.env. In Gate 3, the system automatically creates:

Notion

Database with fields: Title, Status, Role, Effort, Epic, GitHub Issue #. One page per Epic. One task per item — all set to "To Do".

GitHub

Repository with protected main + develop branches. One Issue per task. One Milestone per Epic. Labels: epic names + role names + effort sizes.

During development, every status change syncs automatically:

EventNotion StatusGitHub Action
Task startsIn ProgressBranch created, issue assigned
PR openedIn ReviewPR linked to issue
PR approvedDoneIssue closed, branch deleted

Branch Naming

feature/[issue-#]-[kebab-description]   → new features
fix/[issue-#]-[kebab-description]       → bug fixes
hotfix/[issue-#]-[kebab-description]    → urgent production fixes

Examples:
  feature/3-user-login-screen
  feature/7-call-report-form
  fix/15-login-redirect-bug
  hotfix/23-session-crash
