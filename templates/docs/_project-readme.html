A complete development workflow system for Claude Code. Drop it into ~/.claude/ and get a full software team — 12 specialized AI agents, 17 SDLC workflow runbooks, and battle-tested templates — ready to build any project from scratch.

Quick Start

  1. Clone: git clone https://github.com/clickzika/dev-starter.git
  2. Copy: cp -r dev-starter/* ~/.claude/
  3. Setup: bash ~/.claude/setup.sh
  4. Config: cp ~/.claude/.env.example ~/.claude/.env and fill in tokens
  5. Profile: Edit ~/.claude/USER.md with your skill levels
  6. Start: claude then type Read ~/.claude/dev-menu.md and help me get started

The 5-Gate Build Process

When you start a new project with dev-starter.md, the system walks you through 5 gates. Nothing proceeds without your explicit approval.

GATE 1 Discovery
26 questions (Q1–Q26) covering requirements, tech stack, design preferences, team structure. Produces CLAUDE.md project context.
You say: "Approved"
↓
GATE 2 Architecture & Design
@ba → BRD | @techlead → Architecture | @dba → DB Design | @backend → API | @security → Security | @devops → Infra | @qa → Test Strategy | @uxui → Prototype + Component Library | @pm → Project Plan
You say: "Architecture approved"
↓
GATE 3 Implementation
Sprint-by-sprint coding. Each task: branch → code → test → PR → review → merge. All agents collaborate.
You say: "go" per task
↓
GATE 4 Quality
@qa → Full test suite | @security → Security audit | @techlead → Performance review | All issues resolved before release.
You say: "Quality approved"
↓
GATE 5 Delivery
@devops → Deploy | @docs → Documentation | @pm → Handover | Monitoring configured | Project complete.
You say: "Ship it"

Notion + GitHub Integration

Tokens are stored globally in ~/.claude/.env. During Gate 2–3, the system can automatically create:

Notion

Project database with tasks, statuses, roles, effort estimates. One page per Epic. Syncs with GitHub issues.

GitHub

Repository with branch protection. Issues per task, Milestones per Epic. Labels for roles and effort sizes.
