Overview

DevStarter exposes 24 slash commands as its "API". Each command is a plain-text Markdown file in commands/ that Claude Code loads when the user types /devstarter-X. The command file routes to the matching SDLC runbook in sdlc/ which contains the full workflow logic.

Architecture

commands/devstarter-X.md  →  sdlc/devstarter-X.md  →  agents/devstarter-Y.md

Project Setup Commands

/devstarter-new [description?]
Start a new project from scratch. Runs a 5-gate lifecycle: Discovery → Architecture → Task Setup → Development → Quality & Delivery.
Inline args: yes Gates: 5 Checkpoint: yes
ModeTriggerQuestions
Quick StartPick "1" from picker8 questions
CustomPick "2" from pickerUp to 31 questions
DescribePick "3" or inline arg1 — just describe the project

Example: /devstarter-new React SaaS dashboard with Stripe billing

/devstarter-existing [intent?]
Onboard an existing codebase. Performs a codebase discovery scan, generates CLAUDE.md, and optionally creates missing documents.
Inline args: yes Gates: 1+ Checkpoint: yes
OptionAction
1 — OnboardScan codebase, generate CLAUDE.md, produce discovery report
2 — Add / fixScan + route to /devstarter-change
3 — RefactorScan + plan refactor tasks
4 — SecurityScan + run security audit
5 — Full setupScan + generate all docs + GitHub + Notion
/devstarter-audit [type?]
Audit and review the project across 7 audit types. Produces an audit report and optionally a fix plan.
Inline args: yes Output: docs/audit-report.html

Types: security, performance, code-quality, dependencies, test-coverage, architecture, full

Example: /devstarter-audit security fix → runs security audit and generates a fix plan

Inline Args Convention

Commands that support inline args allow users to bypass the question flow by providing a description directly after the command name. The system extracts intent from the first word(s) and skips all questions that can be inferred.

CommandInline Arg FormatWhat is skipped
/devstarter-new/devstarter-new [project description]All 8–31 intake questions
/devstarter-existing/devstarter-existing [intent]Quick-picker + Q3–Q5 (auto-detected)
/devstarter-change/devstarter-change add [feature]Q1 (operation type) + Q2 (description)
/devstarter-audit/devstarter-audit [type] [outcome]Picker + Q1 + Q5
/devstarter-document/devstarter-document [type]Quick-picker
/devstarter-consult/devstarter-consult [topic]Topic question
