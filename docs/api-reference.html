Overview

DevStarter exposes 63 slash commands as its public API. Since v3.0.0 each command is a SKILL.md file under skills/devstarter-X/ which Claude Code loads when the user types /devstarter-X. The skill file routes to the matching SDLC runbook in sdlc/, which holds the full workflow logic.

Architecture

skills/devstarter-X/SKILL.md  →  sdlc/devstarter-X.md  →  agents/devstarter-Y.md

Project Setup Commands

/devstarter-new [description?]
Start a new project from scratch. Runs a 5-gate lifecycle: Discovery → Architecture → Task Setup → Development → Quality & Delivery. Generates CLAUDE.md, docs, GitHub issues, and Notion workspace.
Inline args: yes Gates: 5 Checkpoint: yes
ModeTriggerQuestions
Quick StartPick "1" from picker8 questions
CustomPick "2" from pickerUp to 31 questions
DescribePick "3" or inline arg1 — describe the project

Example: /devstarter-new React SaaS dashboard with Stripe billing

/devstarter-existing [intent?]
Onboard an existing codebase. Scans the project, generates CLAUDE.md, and optionally creates missing documents or routes to other workflows.
Inline args: yes Gates: 1+ Checkpoint: yes
OptionAction
1 — OnboardScan codebase, generate CLAUDE.md, produce discovery report
2 — Add / fixScan + route to /devstarter-change
3 — RefactorScan + plan refactor tasks
4 — SecurityScan + run security audit
5 — Full setupScan + generate all docs + GitHub labels + Notion board
/devstarter-audit [type?]
Audit the project across 7 dimensions. Produces an HTML audit report and optionally a prioritised fix plan.
Inline args: yes Output: docs/audit-report.html Model: Opus

Types: security · performance · code-quality · dependencies · test-coverage · architecture · full

Example: /devstarter-audit security fix → runs security audit and generates a fix plan.

/devstarter-gitsetup [cleanup?]
Bootstrap git and Gitflow for a project. Creates branches (main/uat/develop), GitHub labels, milestones, PR templates, and optionally applies branch protection rules.
Inline args: yes Phases: 5
Inline ArgBehaviour
(none)Full setup from Phase 1
cleanupSkip to Phase 4.5 — post-merge branch cleanup only
/devstarter-migrate
Stack migration workflow. Analyses the current stack, proposes a target stack, creates a migration plan with rollback checkpoints, and executes phase by phase with gates.
Model: Opus Checkpoint: yes

/devstarter-hotfix
Critical production bug fix. Branches from main, fixes, fast-tracks to production without going through develop. Generates an incident brief (P0/P1) and back-merges to develop after deploy.
Production Model: Opus Checkpoint: yes

Governance & Process

/devstarter-council
Four-voice deliberation for decisions where smart people would disagree. Runs Architect / Skeptic / Pragmatist / Critic in parallel, then synthesises a recommendation.
Model: Opus
VoicePerspective
ArchitectCorrectness, maintainability, long-term health
SkepticChallenges the premise itself
PragmatistShipping speed, ops reality
CriticEdge cases, downside risk, failure modes

Use for: tech selection with real tradeoffs, irreversible architectural decisions, risk assessment under uncertainty. Not for code review → use /devstarter-review.

/devstarter-verification-loop [continuous?]
Run a full 6-phase quality gate on current changes before committing: build → types → lint → tests (80% threshold) → security scan → diff review.
Inline args: yes

continuousRe-run every 15 minutes

Inline Args Convention

Commands that support inline args allow bypassing the question flow by providing a description directly after the command name. The skill extracts intent from the first word(s) and skips all questions that can be inferred.

CommandInline Arg FormatWhat is skipped
/devstarter-new/devstarter-new [project description]All 8–31 intake questions
/devstarter-existing/devstarter-existing [intent]Quick-picker + Q3–Q5 (auto-detected)
/devstarter-change/devstarter-change add [feature]Q1 (operation type) + Q2 (description)
/devstarter-audit/devstarter-audit [type] [outcome]Picker + Q1 + Q5
/devstarter-document/devstarter-document [type]Quick-picker
/devstarter-consult/devstarter-consult [topic]Topic question
/devstarter-debug/devstarter-debug [problem description]Intake question Q1
/devstarter-gitsetup/devstarter-gitsetup cleanupPhases 1–4 (jumps to branch cleanup only)
/devstarter-mcp/devstarter-mcp [server-name]Interactive picker
/devstarter-knowledge/devstarter-knowledge [topic]Topic question
/devstarter-verification-loop/devstarter-verification-loop continuousRun-once mode (switches to loop)
/devstarter-adr/devstarter-adr [decision title]Title question
/devstarter-compliance/devstarter-compliance gdprFramework picker
