Overviewภาพรวม

📝 What you are being asked to approveสิ่งที่ต้องการอนุมัติ

We are extending DevStarter's Obsidian vault integration (v5.8.0 added project snapshots) to cover three more SDLC workflows. The vault will now automatically surface prior patterns when starting a new feature, and offer to capture recurring code-quality findings, audit issues, and retrospective lessons as searchable vault notes. This completes the "knowledge loop" — the vault grows from every workflow, not just from bug postmortems.เราขยาย Obsidian vault integration ของ DevStarter (v5.8.0 เพิ่ม project snapshot) ให้ครอบคลุม workflow SDLC เพิ่มอีก 3 รายการ vault จะแสดง pattern ก่อนหน้าโดยอัตโนมัติเมื่อเริ่ม feature ใหม่ และเสนอให้บันทึก code-quality findings, audit issues, และ retro lessons เป็นโน้ตที่ค้นหาได้

What We Will Buildสิ่งที่จะสร้าง

Four SDLC runbooks receive minimal additive changes — one recall block and three optional emit blocks. No templates, no new note types (reusing existing technique/rca types from v5.7.0).runbook SDLC สี่ไฟล์ได้รับการเปลี่ยนแปลงเพิ่มเติมน้อยที่สุด หนึ่ง recall block และ optional emit สาม block ไม่มี template ใหม่ ไม่มี note type ใหม่ (ใช้ซ้ำ type technique/rca จาก v5.7.0)

  • /devstarter-change (add-feature) — before A-PHASE 2 (Impact Analysis): grep vault for technique notes matching the feature topic. Surfaces prior patterns before design begins./devstarter-change (add-feature) — ก่อน A-PHASE 2: grep vault หา technique notes ที่ตรงกับหัวข้อ feature แสดง pattern ก่อนหน้าก่อนเริ่มออกแบบ
  • /devstarter-review — after Phase 4 (post-review actions): optional vault emit of recurring findings as type: technique, topic: code-quality./devstarter-review — หลัง Phase 4: optional vault emit ของ findings ที่เกิดซ้ำ type: technique, topic: code-quality
  • /devstarter-audit — end of Phase 5 (fix execution): optional vault emit of critical/high findings as type: rca, root_cause_category: audit-gap./devstarter-audit — ท้าย Phase 5: optional vault emit ของ findings ระดับ critical/high type: rca, root_cause_category: audit-gap
  • /devstarter-retro — after Phase 4 (action items created): optional vault emit of process lessons as type: technique, topic: process./devstarter-retro — หลัง Phase 4: optional vault emit ของ process lessons type: technique, topic: process
User StoryUser Story

As a developer starting a new feature, I want DevStarter to surface prior vault notes matching my feature topic — so I don't repeat solved problems. As a reviewer, I want to capture recurring findings to the vault — so patterns are shared across projects without manual effort.ในฐานะนักพัฒนาที่เริ่ม feature ใหม่ ฉันต้องการให้ DevStarter แสดง vault notes ก่อนหน้าที่ตรงกับหัวข้อ feature เพื่อไม่ต้องแก้ปัญหาซ้ำ ในฐานะ reviewer ฉันต้องการบันทึก recurring findings ลงใน vault เพื่อแชร์ pattern ข้ามโปรเจกต์โดยอัตโนมัติ

✖ Out of Scopeอยู่นอกขอบเขต

Acceptance Criteriaเกณฑ์การยอมรับ

  • Given obsidian.enabled:true, When /devstarter-change (add-feature) reaches A-PHASE 2, Then vault is grepped for technique notes matching the feature topic and top matches surfaced.เมื่อ obsidian.enabled:true เมื่อ /devstarter-change (add-feature) ถึง A-PHASE 2 จะ grep vault หา technique notes ที่ตรงกับหัวข้อ feature
  • Given /devstarter-review completes Phase 4, When obsidian.enabled:true, Then optional "Save recurring findings to vault?" gate offered; if yes, emits type: technique notes.เมื่อ /devstarter-review เสร็จ Phase 4 เมื่อ obsidian.enabled:true จะเสนอ gate "บันทึก findings ลงใน vault?"
  • Given /devstarter-audit Phase 5 complete, When obsidian.enabled:true, Then optional emit of critical/high findings as type: rca notes.เมื่อ /devstarter-audit Phase 5 เสร็จ เมื่อ obsidian.enabled:true จะเสนอ emit findings ระดับ critical/high
  • Given /devstarter-retro Phase 4 complete, When obsidian.enabled:true, Then optional emit of process lessons as type: technique, topic: process.เมื่อ /devstarter-retro Phase 4 เสร็จ เมื่อ obsidian.enabled:true จะเสนอ emit process lessons
  • When obsidian.enabled:false, all four workflows behave exactly as before (silently skip).เมื่อ obsidian.enabled:false ทุก workflow ทำงานเหมือนเดิม

Management Summaryสรุปสำหรับผู้บริหาร

Why nowทำไมต้องทำตอนนี้

v5.8.0 gave the vault a "welcome to this project" note. v5.9.0 completes the knowledge loop: patterns from code reviews, audit findings, and retrospective lessons now flow into the vault automatically, making the shared knowledge base grow from every workflow instead of requiring manual capture.v5.8.0 เพิ่มโน้ต "ยินดีต้อนรับสู่โปรเจกต์" ใน vault v5.9.0 ทำ knowledge loop สมบูรณ์: pattern จาก code review, audit findings, และ retro lessons ไหลเข้า vault โดยอัตโนมัติ

Who benefitsใครได้ประโยชน์

All developers using DevStarter — especially teams where the same code-quality issues recur across projects. Prior fix patterns surface before new features are designed, preventing repeat mistakes.นักพัฒนาทุกคนที่ใช้ DevStarter โดยเฉพาะทีมที่มีปัญหา code-quality ซ้ำข้ามโปรเจกต์

Cost of not doing thisผลกระทบหากไม่ดำเนินการ

Teams continue to repeat known mistakes. The vault remains a postmortem archive rather than a living knowledge base. The value of the Obsidian investment from v5.7.0 is not fully realized.ทีมยังทำผิดซ้ำ vault ยังคงเป็นเพียง postmortem archive ไม่ใช่ knowledge base ที่มีชีวิต
