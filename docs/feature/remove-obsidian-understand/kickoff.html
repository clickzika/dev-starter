Overviewภาพรวม

📝 What you are being asked to approve

Remove two complete feature sets from DevStarter: the Obsidian Knowledge Vault integration (v5.7.0–v6.0.0) and the Understand-Anything plugin wrappers (v5.6.0). This deletes all related skills, SDLC runbooks, templates, and docs, and strips vault/understand references from 9 other runbooks. No production system or user data is affected — DevStarter is a Markdown toolkit. นำฟีเจอร์สองชุดออกจาก DevStarter ได้แก่ การผสานรวม Obsidian Knowledge Vault (v5.7.0–v6.0.0) และ wrapper ของ plugin Understand-Anything (v5.6.0) โดยลบ skill, SDLC runbook, template และเอกสารที่เกี่ยวข้องทั้งหมด รวมทั้งลบการอ้างอิง vault/understand ออกจาก runbook อื่นอีก 9 รายการ ไม่มีผลกระทบต่อระบบ production หรือข้อมูลผู้ใช้ เนื่องจาก DevStarter เป็น Markdown toolkit

What We Will Removeสิ่งที่เราจะนำออก

👤 For the requester — please confirm this matches your intentสำหรับผู้ร้องขอ — โปรดยืนยันว่าตรงกับความต้องการ

Two feature groups built into DevStarter between v5.6.0 and v6.0.0 will be completely removed. The Obsidian Knowledge Vault added commands for capturing bugs, techniques, RCAs, and project snapshots into an Obsidian vault shared across projects. The Understand-Anything wrappers added 8 commands that delegated codebase analysis to an external plugin. Both features are being removed because they are out of scope for the toolkit's core value proposition. ฟีเจอร์สองกลุ่มที่สร้างใน DevStarter ระหว่าง v5.6.0 ถึง v6.0.0 จะถูกนำออกทั้งหมด กลุ่ม Obsidian Knowledge Vault เพิ่ม command สำหรับบันทึก bug, technique, RCA และ project snapshot ลงใน Obsidian vault ที่แชร์ข้ามโปรเจกต์ กลุ่ม Understand-Anything wrappers เพิ่ม command 8 รายการที่ส่งต่อการวิเคราะห์ codebase ไปยัง plugin ภายนอก ทั้งสองฟีเจอร์ถูกนำออกเนื่องจากอยู่นอกขอบเขตหลักของ toolkit

Rollback Planแผนการย้อนกลับ

All changes are on branch chore/remove-obsidian-understand. To restore either feature, revert the branch or cherry-pick the removal commit. No database migrations, no deployed services, no external state to undo — a git revert is sufficient. การเปลี่ยนแปลงทั้งหมดอยู่บน branch chore/remove-obsidian-understand หากต้องการกู้คืนฟีเจอร์ใด ให้ revert branch หรือ cherry-pick commit ที่ลบออก ไม่มี database migration, ไม่มี service ที่ deploy, ไม่มี state ภายนอก — git revert เพียงพอ

Management Summaryสรุปสำหรับผู้บริหาร

📊 For management — business rationale, no technical detailสำหรับผู้บริหาร — เหตุผลเชิงธุรกิจ ไม่มีรายละเอียดทางเทคนิค

Why now — the business needทำไมต้องทำตอนนี้ — ความจำเป็นทางธุรกิจ

The Obsidian and Understand-Anything features were added to expand DevStarter's scope but have been determined to be outside the toolkit's core focus — helping teams ship software faster using structured AI workflows. Removing them simplifies the product, reduces maintenance surface, and keeps the command list focused on the SDLC operations that provide the most value. ฟีเจอร์ Obsidian และ Understand-Anything ถูกเพิ่มเพื่อขยายขอบเขตของ DevStarter แต่พบว่าอยู่นอกเหนือจากจุดเน้นหลักของ toolkit ซึ่งคือการช่วยทีม ship software ได้เร็วขึ้นด้วย AI workflow ที่มีโครงสร้าง การนำออกช่วยให้ผลิตภัณฑ์เรียบง่ายขึ้น ลด surface area ที่ต้องบำรุงรักษา และทำให้รายการ command มุ่งเน้นที่การดำเนินงาน SDLC ที่ให้คุณค่าสูงสุด

Who benefitsใครได้ประโยชน์

All DevStarter users benefit from a leaner, faster-to-navigate toolkit. New users onboarding face fewer commands to learn. Maintainers have less surface to keep up to date. ผู้ใช้ DevStarter ทุกคนได้ประโยชน์จาก toolkit ที่กระชับและนำทางง่ายขึ้น ผู้ใช้ใหม่ที่เริ่มใช้งานมี command น้อยลงต้องเรียนรู้ ผู้ดูแลมี surface area น้อยลงที่ต้องอัปเดต

Cost of not doing thisผลกระทบหากไม่ดำเนินการ

Keeping these features means continued maintenance overhead for two feature sets that are not being actively used or developed. The vault integration in particular has 9 integration points that must be kept consistent whenever other SDLC runbooks are updated. การคงฟีเจอร์เหล่านี้หมายถึงภาระการบำรุงรักษาต่อเนื่องสำหรับฟีเจอร์สองชุดที่ไม่ได้ใช้งานหรือพัฒนาอย่างจริงจัง การผสานรวม vault โดยเฉพาะมีจุดเชื่อมต่อ 9 จุดที่ต้องคงความสอดคล้องทุกครั้งที่มีการอัปเดต SDLC runbook อื่น
