Root Problem / Requirementต้นตอของปัญหา / ความต้องการ

Original Requirementความต้องการเดิม

DevStarter accumulated two optional integration layers over v5.6.0–v6.0.0: (1) Understand-Anything wrappers — 8 thin /devstarter-understand* commands that delegated to an external Understand-Anything plugin, requiring co-installation and preflight plugin detection; (2) Obsidian Knowledge Vault — 2 commands (/devstarter-knowledge, /devstarter-vault-ingest), 4 note templates, and 9 emit/recall integration hooks wired into core SDLC runbooks (change-add, debug, postmortem, review, retro, audit, onboarding, starter-gates, release-verify, bug-postmortem). Both features required external dependencies not present in most installs and added cognitive overhead to the base toolkit without being part of the core value proposition.DevStarter สะสมการผสานรวมแบบ optional สองชั้นในช่วง v5.6.0–v6.0.0: (1) Understand-Anything wrappers — คำสั่ง /devstarter-understand* แบบบาง 8 คำสั่งที่มอบหมายไปยัง plugin Understand-Anything ภายนอก ต้องติดตั้งร่วมกันและตรวจสอบ plugin ก่อน; (2) Obsidian Knowledge Vault — 2 คำสั่ง, เทมเพลตโน้ต 4 รายการ และ hooks การผสานรวม emit/recall 9 รายการ ที่เชื่อมต่อกับ SDLC runbooks หลัก ทั้งสองฟีเจอร์ต้องการ dependencies ภายนอกที่ส่วนใหญ่ไม่มีในการติดตั้ง และเพิ่มภาระทางความคิดให้กับ toolkit พื้นฐานโดยไม่ได้เป็นส่วนหนึ่งของคุณค่าหลัก

Business Gapช่องว่างทางธุรกิจ

Both feature sets depended on external tools (a configured Obsidian vault path, a co-installed Understand-Anything plugin) that the majority of DevStarter users do not have. The 9 SDLC runbook emit/recall blocks added silent no-ops for all users without a configured vault, and the 8 understand wrappers required a plugin that most users never installed. Keeping them in core increased install surface area, added menu clutter (10 extra entries), and introduced maintenance burden with no active user base for either integration.ฟีเจอร์ทั้งสองชุดขึ้นอยู่กับเครื่องมือภายนอก (path ของ Obsidian vault ที่กำหนดค่าไว้, plugin Understand-Anything ที่ติดตั้งร่วม) ซึ่งผู้ใช้ DevStarter ส่วนใหญ่ไม่มี บล็อก emit/recall ใน SDLC runbooks 9 รายการเพิ่ม no-ops แบบเงียบสำหรับผู้ใช้ทุกคนที่ไม่มี vault ที่กำหนดค่าไว้ และ 8 wrappers ต้องการ plugin ที่ผู้ใช้ส่วนใหญ่ไม่เคยติดตั้ง การเก็บไว้ใน core เพิ่มพื้นที่ติดตั้ง, เพิ่มความยุ่งเหยิงในเมนู (10 รายการพิเศษ) และสร้างภาระการบำรุงรักษาโดยไม่มีฐานผู้ใช้งานจริงสำหรับการผสานรวมทั้งสองอย่าง

How This Change Resolves Itการเปลี่ยนแปลงนี้แก้ไขอย่างไร

Complete hard deletion of both integration layers: 10 skills deleted, 10 SDLC runbooks deleted, 4 Obsidian note templates deleted, 2 ADRs deleted, all associated docs/feature subdirectories deleted, 9 SDLC runbook vault/recall blocks stripped, devstarter-menu.md and devstarter-config.yml cleaned. DevStarter is now self-contained with no external tool dependencies baked into core commands.ลบชั้นการผสานรวมทั้งสองอย่างอย่างสมบูรณ์: ลบ skills 10 รายการ, ลบ SDLC runbooks 10 รายการ, ลบเทมเพลตโน้ต Obsidian 4 รายการ, ลบ ADRs 2 รายการ, ลบ subdirectories docs/feature ที่เกี่ยวข้องทั้งหมด, ลบบล็อก vault/recall ออกจาก SDLC runbooks 9 รายการ, ทำความสะอาด devstarter-menu.md และ devstarter-config.yml DevStarter ตอนนี้เป็น self-contained โดยไม่มี dependencies เครื่องมือภายนอกที่ฝังอยู่ใน core commands

Implementation Notesบันทึกการพัฒนา

Key Decisions Madeการตัดสินใจสำคัญ

  • Hard delete, no deprecation path. Both features were optional integrations with no measurable user base. A deprecation notice would add noise without benefit.ลบโดยตรง ไม่มีขั้นตอน deprecation ทั้งสองฟีเจอร์เป็น optional integrations ที่ไม่มีฐานผู้ใช้งานที่วัดได้ การแจ้ง deprecation จะเพิ่มความยุ่งเหยิงโดยไม่มีประโยชน์
  • ADRs deleted alongside features. ADR-0001 and ADR-0002 document decisions for features that no longer exist; retaining orphaned ADRs would be misleading.ลบ ADRs พร้อมกับฟีเจอร์ ADR-0001 และ ADR-0002 บันทึกการตัดสินใจสำหรับฟีเจอร์ที่ไม่มีอยู่แล้ว การเก็บ ADRs ที่เป็น orphan ไว้จะทำให้เข้าใจผิด
  • Verification grep after edit pass. After the initial 9-runbook edit list was processed, a grep -r "obsidian|vault|understand" sweep across skills/ and sdlc/ was run. It surfaced two additional files not in the original plan (devstarter-bug-postmortem/SKILL.md, devstarter-release-verify.md) — both were fixed before commit.ตรวจสอบด้วย grep หลัง edit pass หลังจากประมวลผลรายการ runbook เดิม 9 รายการแล้ว ได้รัน grep -r ครอบคลุม skills/ และ sdlc/ พบไฟล์เพิ่มเติม 2 ไฟล์ที่ไม่อยู่ในแผนเดิม — ทั้งคู่ได้รับการแก้ไขก่อน commit
  • Untracked consult files removed with rm -f. Five memory/consult-*.md files were not in the git index (never staged). Used rm -f directly instead of git rm.ไฟล์ consult ที่ไม่ได้ track ถูกลบด้วย rm -f ไฟล์ memory/consult-*.md 5 ไฟล์ไม่ได้อยู่ใน git index ใช้ rm -f โดยตรงแทน git rm
  • settings.json and .project.env excluded from removal commit. Both had pre-existing unrelated staged changes; kept out of this commit to maintain clean commit history.ยกเว้น settings.json และ .project.env จาก removal commit ทั้งคู่มีการเปลี่ยนแปลงที่ staged ไว้แล้วซึ่งไม่เกี่ยวข้องกัน; เก็บไว้นอก commit นี้เพื่อรักษาประวัติ commit ที่สะอาด

Technical Approachแนวทางเชิงเทคนิค

Two-pass removal: (1) Hard delete all files unique to the removed features using git rm -r --force for tracked files and rm -f for untracked ones. (2) Strip vault/understand integration blocks from reference files using the Edit tool — each block identified by content pattern, excised with surrounding whitespace to leave clean flow. Post-edit verification grep (grep -r "obsidian\|vault\|understand" skills/ sdlc/) confirmed zero remaining references before committing.การลบสองรอบ: (1) ลบไฟล์ที่เป็น unique ของฟีเจอร์ที่ลบออกทั้งหมดโดยใช้ git rm -r --force สำหรับไฟล์ที่ tracked และ rm -f สำหรับไฟล์ที่ไม่ได้ tracked (2) ลบบล็อก vault/understand integration ออกจากไฟล์อ้างอิงโดยใช้ Edit tool — แต่ละบล็อกระบุด้วย content pattern ลบพร้อมกับ whitespace โดยรอบเพื่อให้ flow สะอาด การตรวจสอบด้วย grep หลัง edit ยืนยัน zero references ที่เหลืออยู่ก่อน commit
