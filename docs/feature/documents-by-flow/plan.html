Problem / Requirement

Root Problem

No single place documents which artifacts each flow emits.ไม่มีที่เดียวที่ระบุว่าแต่ละโฟลว์สร้างเอกสารใด

Business Requirement

A discoverable reference in the README.คู่มืออ้างอิงที่ค้นเจอได้ใน README

User Story

As a DevStarter userผู้ใช้ DevStarter,
I want to a per-flow document referenceคู่มือเอกสารต่อโฟลว์,
So that know expected artifacts up frontรู้เอกสารที่จะได้ล่วงหน้า.

Acceptance Criteria

  • Given the README, When a reader scans it, Then a 'Documents by Flow' section existsเมื่อเปิด README จะพบส่วน 'Documents by Flow'
  • All five flows are listed with their plain + technical docsระบุครบทั้งห้าโฟลว์พร้อมเอกสารภาษาคนและเชิงเทคนิค
3

Proposed Solution

Approach

Add a markdown table section to README mapping flows to documents.เพิ่มตารางใน README จับคู่โฟลว์กับเอกสาร

Why This Approach

README is the canonical entry point; a table is scannable and low-maintenance.README เป็นจุดเริ่มหลัก ตารางอ่านง่ายและดูแลรักษาง่าย

Alternatives Considered

OptionReason Rejected
Separate docs pageหน้าเอกสารแยกRejected — extra file to maintain, less discoverableปฏิเสธ — มีไฟล์เพิ่มต้องดูแล ค้นเจอยากกว่า
