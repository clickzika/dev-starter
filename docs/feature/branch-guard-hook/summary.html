Root Problem / Requirementปัญหาหลัก / ความต้องการ

Original Problem / Requirementปัญหา / ความต้องการเดิม

Branch Guard (Rule 9) existed as text rules in devstarter-change.md and agents/shared/devstarter-agent-base.md only. The rule instructed Claude to check the branch before editing, but no technical mechanism enforced it. The AI model could still write files on protected branches (develop, main, uat) during autopilot mode, resume flows, or edge cases where the rule was overlooked.Branch Guard (Rule 9) มีเพียงกฎข้อความใน devstarter-change.md และ agents/shared/devstarter-agent-base.md เท่านั้น กฎสั่งให้ Claude ตรวจสอบ branch ก่อนแก้ไข แต่ไม่มีกลไกทางเทคนิคบังคับใช้ AI model ยังสามารถเขียนไฟล์บน protected branch (develop, main, uat) ในโหมด autopilot, resume flow หรือกรณีพิเศษที่มองข้ามกฎ

Root Cause / Business Gapสาเหตุหลัก / ช่องว่างทางธุรกิจ

No Claude Code PreToolUse hook existed to intercept Edit/Write tool calls before they executed. Enforcement was 100% documentation-based with no technical safety net. This meant a single missed rule check could result in commits directly on a protected branch.ไม่มี PreToolUse hook ของ Claude Code ที่สกัดกั้นการเรียก tool Edit/Write ก่อนทำงาน การบังคับใช้พึ่งพาเอกสาร 100% โดยไม่มีตาข่ายความปลอดภัยทางเทคนิค ซึ่งหมายความว่าการตรวจสอบกฎที่พลาดไปครั้งเดียวอาจส่งผลให้มีการ commit โดยตรงบน protected branch

How This Change Resolves Itการแก้ไขปัญหาด้วยการเปลี่ยนแปลงนี้

Added scripts/hooks/pre-edit-branch-guard.js — a PreToolUse hook that runs before every Edit or Write tool call. It checks git branch --show-current and outputs a JSON block decision ({"decision":"block","reason":"..."}) if Claude is on a protected branch. The block message names the current branch and instructs Claude to create the correct work branch. Non-git directories pass through without error.เพิ่ม scripts/hooks/pre-edit-branch-guard.js — PreToolUse hook ที่ทำงานก่อนการเรียก tool Edit หรือ Write ทุกครั้ง ตรวจสอบ git branch --show-current และส่งออกคำตัดสินบล็อก JSON ({"decision":"block","reason":"..."}) หาก Claude อยู่บน protected branch ข้อความบล็อกระบุชื่อ branch ปัจจุบันและแนะนำให้ Claude สร้าง work branch ที่ถูกต้อง ไดเรกทอรีที่ไม่ใช่ git ผ่านได้โดยไม่มีข้อผิดพลาด

Implementation Notesหมายเหตุการพัฒนา

Key Decisions Madeการตัดสินใจสำคัญ

  • Used {"decision":"block","reason":"..."} stdout JSON format (Claude Code PreToolUse block protocol) rather than non-zero exit, so the error message is human-readable in Claude's contextใช้รูปแบบ JSON stdout {"decision":"block","reason":"..."} (Claude Code PreToolUse block protocol) แทนการออกด้วย non-zero เพื่อให้ข้อความผิดพลาดอ่านได้ใน context ของ Claude
  • Exact match on protected branch names (no glob/prefix) to avoid false blocks on branches like develop-v2จับคู่แบบ exact กับชื่อ protected branch (ไม่ใช้ glob/prefix) เพื่อหลีกเลี่ยงการบล็อกเท็จบน branch เช่น develop-v2
  • git rev-parse --git-dir called first to detect non-git dirs; any exception causes pass-through to prevent false blocksเรียก git rev-parse --git-dir ก่อนเพื่อตรวจจับ non-git dir; exception ใดๆ ทำให้ผ่านเพื่อป้องกันการบล็อกเท็จ
  • Block message is educational: names the current branch and tells Claude exactly which branch type to createข้อความบล็อกเป็นการสอน: ระบุชื่อ branch ปัจจุบันและบอก Claude ว่าต้องสร้าง branch ประเภทใด
  • Hook registered in templates/hooks/hooks.json (not directly in .claude/settings.json) so it deploys correctly via install.sh --hooksHook ลงทะเบียนใน templates/hooks/hooks.json (ไม่ใช่โดยตรงใน .claude/settings.json) เพื่อให้ deploy ได้ถูกต้องผ่าน install.sh --hooks

Technical Approachแนวทางเทคนิค

The hook follows the same Node.js stdin/stdout pass-through pattern as stop-check-console-log.js. It reads stdin (up to 64KB), runs two synchronous execFileSync git calls (rev-parse to detect git repo, then branch --show-current), checks the result against a Set of protected branch names, and either outputs a block decision JSON or passes stdin through unchanged. The hook always exits 0 — Claude Code handles the block via the JSON decision field, not exit codes.Hook ตามรูปแบบ Node.js stdin/stdout pass-through เดียวกันกับ stop-check-console-log.js อ่าน stdin (สูงสุด 64KB), รัน git calls แบบ synchronous สองครั้ง (rev-parse เพื่อตรวจจับ git repo, จากนั้น branch --show-current), ตรวจสอบผลลัพธ์กับ Set ของชื่อ protected branch และส่งออกคำตัดสินบล็อก JSON หรือผ่าน stdin โดยไม่เปลี่ยนแปลง Hook ออกด้วย 0 เสมอ — Claude Code จัดการบล็อกผ่านฟิลด์คำตัดสิน JSON ไม่ใช่ exit code

Deviations from Planความเบี่ยงเบนจากแผน

Hook logic, protected branch list, pass-through on non-git, and educational block message all match the consultation and plan.html spec exactly.ไม่มี — พัฒนาตามแผน ตรรกะ hook, รายการ protected branch, การผ่านบน non-git และข้อความบล็อกแบบสอนตรงกับข้อกำหนดในการปรึกษาและ plan.html ทุกประการ
