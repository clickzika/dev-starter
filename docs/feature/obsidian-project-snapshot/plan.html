Problem & Business Requirementปัญหา & ความต้องการ

Root gapช่องว่างหลัก

DevStarter v5.7.0 ships a vault that captures bugs and root causes, but has no project-level note type. Team members joining a project have no vault entry point — they must read CLAUDE.md manually or ask the team. The vault's onboarding value is zero until this is fixed.DevStarter v5.7.0 มี vault สำหรับบันทึก bug และต้นตอของปัญหา แต่ไม่มีประเภทโน้ตระดับโปรเจกต์ สมาชิกทีมใหม่ไม่มีจุดเริ่มต้นใน vault ต้องอ่าน CLAUDE.md เอง ทำให้คุณค่าของ vault ต่อการ onboarding เป็นศูนย์จนกว่าจะแก้ปัญหานี้

User storyUser story

As a developer joining a new project, I want to open the knowledge vault and immediately find a project-snapshot note showing the tech stack, key decisions, and constraints — so that I can orient myself without interrupting the team.ในฐานะนักพัฒนาที่เข้าร่วมโปรเจกต์ใหม่ ฉันต้องการเปิด knowledge vault แล้วพบโน้ต project-snapshot ที่แสดง tech stack การตัดสินใจหลัก และข้อจำกัดทันที เพื่อให้ฉันปรับตัวได้เองโดยไม่ต้องรบกวนทีม

Acceptance criteriaเกณฑ์การยอมรับ

  • Given obsidian.enabled: true, When /devstarter-new Phase 5 completes, Then optional project-snapshot note emitted (sanitized, with type/stack/key-decisions/repo-url).เมื่อ obsidian.enabled: true เมื่อ /devstarter-new Phase 5 เสร็จ จะ emit โน้ต project-snapshot (ผ่าน sanitize พร้อม type/stack/key-decisions/repo-url)
  • Given a project-snapshot in vault, When /devstarter-onboard runs, Then vault is grepped and note surfaced as orientation entry.เมื่อมี project-snapshot ใน vault เมื่อ /devstarter-onboard ทำงาน จะ grep vault และแสดงโน้ตเป็นข้อมูลปฐมนิเทศ
  • Given obsidian.enabled: true, When /devstarter-release Phase 9.5 runs, Then versioned snapshot emitted (unique filename, version key added).เมื่อ obsidian.enabled: true เมื่อ /devstarter-release Phase 9.5 ทำงาน จะ emit snapshot เวอร์ชัน (ชื่อไฟล์ไม่ซ้ำ)
  • When obsidian.enabled: false, all three workflows behave exactly as before (emit silently skipped).เมื่อ obsidian.enabled: false ทั้งสาม workflow ทำงานเหมือนเดิม (ข้าม emit โดยอัตโนมัติ)

Solution Approachแนวทางแก้ปัญหา

Chosen approachแนวทางที่เลือก

Add a new type: project-snapshot note template that is emitted at natural project lifecycle milestones. The template extends the existing Vault Emit Procedure (E1–E5) and Frontmatter Schema already defined in sdlc/devstarter-knowledge.md. Three runbooks get minimal additive changes — a gated optional emit block each. No existing behavior changes.เพิ่ม template โน้ต type: project-snapshot ใหม่ที่ emit ตาม lifecycle ของโปรเจกต์ที่เหมาะสม template นี้ใช้ Vault Emit Procedure (E1–E5) และ Frontmatter Schema ที่มีอยู่แล้วใน sdlc/devstarter-knowledge.md runbook สามไฟล์ได้รับการเปลี่ยนแปลงเพิ่มเติมน้อยที่สุด คือ optional emit block ที่ผ่าน gate แต่ละอัน โดยไม่มีการเปลี่ยนแปลงพฤติกรรมเดิม
