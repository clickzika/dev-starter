Problem / Requirementปัญหา / ความต้องการ

Root Problemต้นตอของปัญหา

Step I1 of sdlc/devstarter-vault-ingest.md performs a hard-stop for any file that is not .md: "If not found or not .md → stop and tell the user." No HTML extraction layer exists between the raw HTML input and the classify/frontmatter flow (Steps I2+). The runbook was originally designed for Markdown-only ingest and HTML support was never added.Step I1 ใน sdlc/devstarter-vault-ingest.md หยุดทำงานทันทีสำหรับไฟล์ที่ไม่ใช่ .md: "If not found or not .md → stop and tell the user." ไม่มีชั้นดึงข้อมูล HTML ระหว่าง input HTML กับกระบวนการ classify/frontmatter (Steps I2+) รันบุ๊กถูกออกแบบมาสำหรับ Markdown-only ingest และไม่เคยเพิ่มการรองรับ HTML

Business Requirementความต้องการทางธุรกิจ

DevStarter's docs/ directory contains 37 HTML files generated by SDLC workflows: feature kickoffs, plans, summaries, management briefs, ADRs, and release notes. All are generated by DevStarter itself using a consistent <section>/<h1>/<h2>/<pre> template structure. These files should be ingestible into the Obsidian vault as searchable Markdown notes using the same /devstarter-vault-ingest command already used for .md files.โฟลเดอร์ docs/ ของ DevStarter มีไฟล์ HTML 37 ไฟล์ที่สร้างโดย SDLC workflows: feature kickoffs, plans, summaries, management briefs, ADRs และ release notes ทั้งหมดสร้างโดย DevStarter เองโดยใช้โครงสร้าง template <section>/<h1>/<h2>/<pre> ที่สม่ำเสมอ ไฟล์เหล่านี้ควรนำเข้า Obsidian vault เป็น Markdown notes ที่ค้นหาได้โดยใช้คำสั่ง /devstarter-vault-ingest เดิมที่ใช้กับไฟล์ .md อยู่แล้ว

As a DevStarter user,
I want to run /devstarter-vault-ingest on a .html file from docs/,
So that a sanitized vault note is created with the document's title and content — without manually converting the file to Markdown.ในฐานะผู้ใช้ DevStarter,

Proposed Solutionแนวทางแก้ไขที่เสนอ

Approach — Option A: Model-native HTML Extractionแนวทาง — ตัวเลือก A: Model-native HTML Extraction

Add a new preprocessing step (Step I1b) to the vault-ingest runbook. When the input file is .html or .htm, the model reads the file and extracts clean Markdown using the following rules, then feeds the result into the existing I2–I11 flow as if it were a .md source file:เพิ่มขั้นตอน preprocessing ใหม่ (Step I1b) ใน vault-ingest runbook เมื่อไฟล์ input เป็น .html หรือ .htm model จะอ่านไฟล์และดึงเนื้อหา Markdown ที่สะอาดโดยใช้กฎต่อไปนี้ จากนั้นส่งผลลัพธ์เข้าสู่ flow I2–I11 ที่มีอยู่เหมือนกับว่าเป็นไฟล์ .md 

Extraction rules for Step I1b:
1. Title     → first <h1> content OR <title> text (strip " — DevStarter" suffix if present)
2. Sections  → each <section>/<main>/<article> → ## heading + body
3. Headings  → <h2> → ##  |  <h3> → ###
4. Tables    → <table> → GFM pipe table
5. Code      → <pre><code class="lang-X"> → ``` X fenced block
6. Paragraphs→ <p> text → plain paragraph
7. Strip     → <nav>, <header>, <footer>, <script>, <style>, <aside>, all HTML attributes
8. Compose   → clean MD string → feed to I2 (classify) as if it were the source .md content

Why This Approachเหตุผลที่เลือกแนวทางนี้

DevStarter's architecture constraint is bash + markdown only, no external tools, install.sh distribution. Adding pandoc or markdownify as system dependencies breaks this contract entirely. All 37 target HTML files are DevStarter-generated with predictable structure — model extraction is reliable for this known template. Option A requires zero new system dependencies.ข้อจำกัดทางสถาปัตยกรรมของ DevStarter คือ bash + markdown เท่านั้น ไม่มีเครื่องมือภายนอก ใช้ install.sh สำหรับแจกจ่าย การเพิ่ม pandoc หรือ markdownify เป็น system dependencies จะทำลาย contract นี้โดยสิ้นเชิง ไฟล์ HTML เป้าหมายทั้ง 37 ไฟล์สร้างโดย DevStarter ที่มีโครงสร้างที่คาดเดาได้ ทำให้ model extraction เชื่อถือได้สำหรับ template ที่รู้จักนี้ ตัวเลือก A ไม่ต้องการ system dependencies ใหม่เลย

OptionReason Rejected

Adds pandoc/Python as install requirements; incompatible with bash install.sh-only distribution model.ทำลายข้อจำกัด "ไม่มี external deps" ต้องติดตั้ง pandoc/Python เพิ่มเติม ไม่เข้ากันกับโมเดลการแจกจ่ายแบบ bash install.sh
