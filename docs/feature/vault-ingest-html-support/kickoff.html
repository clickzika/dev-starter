Overviewภาพรวม

📝 What you are being asked to approve

Extend the /devstarter-vault-ingest skill to accept .html and .htm files in addition to the existing .md files. The skill will extract clean Markdown content from the HTML (title, headings, paragraphs, tables, code blocks) and store it as a sanitized note in the Obsidian vault — with no new system dependencies required.ขยายความสามารถของ /devstarter-vault-ingest เพื่อรองรับไฟล์ .html และ .htm นอกเหนือจากไฟล์ .md ที่มีอยู่แล้ว ระบบจะดึงเนื้อหา Markdown ที่สะอาดออกจาก HTML (ชื่อเรื่อง หัวข้อ ย่อหน้า ตาราง และโค้ด) แล้วจัดเก็บเป็น note ที่ผ่านการ sanitize ใน Obsidian vault โดยไม่ต้องติดตั้งเครื่องมือเพิ่มเติม

What We Will Changeสิ่งที่เราจะเปลี่ยน

👤 For the requester — please confirm this matches your intentสำหรับผู้ร้องขอ — โปรดยืนยันว่าตรงกับความต้องการ

Currently, /devstarter-vault-ingest only accepts .md files. Step I1 of the runbook performs a hard-stop for any other file extension. DevStarter's docs/ directory contains 37 HTML files (feature docs, ADRs, release notes) that cannot be ingested into the Obsidian vault under the current rules.ปัจจุบัน /devstarter-vault-ingest รองรับเฉพาะไฟล์ .md เท่านั้น ขั้นตอน I1 ในรันบุ๊กจะหยุดทันทีเมื่อพบไฟล์นามสกุลอื่น โดยในโฟลเดอร์ docs/ ของ DevStarter มีไฟล์ HTML 37 ไฟล์ (เอกสาร feature, ADR, release notes) ที่ยังไม่สามารถนำเข้า vault ได้ภายใต้กฎปัจจุบัน

We will modify Step I1 to accept .html/.htm files, and add a new preprocessing step (Step I1b) that extracts clean Markdown from the HTML structure before feeding it into the existing classify→frontmatter→emit flow (Steps I2–I11). All existing behaviour for .md files is unchanged.เราจะปรับปรุง Step I1 ให้รองรับไฟล์ .html/.htm และเพิ่มขั้นตอนใหม่ (Step I1b) สำหรับดึงเนื้อหา Markdown จากโครงสร้าง HTML ก่อนส่งเข้าสู่กระบวนการจัดประเภท→ frontmatter→ emit ที่มีอยู่แล้ว (Steps I2–I11) พฤติกรรมสำหรับไฟล์ .md ทั้งหมดไม่มีการเปลี่ยนแปลง

User Storyเรื่องราวของผู้ใช้

As a DevStarter user with HTML feature docs in docs/,
I want to run /devstarter-vault-ingest docs/feature/X/plan.html,
So that a sanitized vault note is created with the document's title and body — without having to manually convert the file to Markdown first.ในฐานะ ผู้ใช้ DevStarter ที่มีเอกสาร HTML ใน docs/,
ฉันต้องการ รันคำสั่ง /devstarter-vault-ingest docs/feature/X/plan.html,
เพื่อที่ vault note ที่ผ่านการ sanitize จะถูกสร้างขึ้นพร้อมชื่อเรื่องและเนื้อหาของเอกสาร โดยไม่ต้องแปลงไฟล์เป็น Markdown ด้วยตนเองก่อน

Scope & Acceptance Criteriaขอบเขต & เกณฑ์การยอมรับ

✓ In Scopeอยู่ในขอบเขต
  • Accept .html and .htm file extensions in Step I1รองรับนามสกุล .html และ .htm ใน Step I1
  • Add Step I1b: extract title, headings, paragraphs, tables, code blocks; strip nav/header/footer/script/style/asideเพิ่ม Step I1b: ดึงชื่อเรื่อง หัวข้อ ย่อหน้า ตาราง โค้ด; ลบ nav/header/footer/script/style/aside
  • E4 sanitize still applied on extracted HTML content before writeE4 sanitize ยังคงทำงานกับเนื้อหาที่ดึงออกมาจาก HTML ก่อนเขียน
  • Update SKILL.md inline args to mention HTML supportอัปเดต SKILL.md เพื่อระบุว่ารองรับ HTML
✖ Out of Scopeอยู่นอกขอบเขต
  • External tool dependencies (pandoc, markdownify, Python)การพึ่งพาเครื่องมือภายนอก (pandoc, markdownify, Python)
  • General-purpose HTML conversion for arbitrary third-party pagesการแปลง HTML ทั่วไปสำหรับหน้าเว็บของบุคคลที่สาม
  • Changes to Steps I2–I11 (classify, frontmatter, emit flow)การเปลี่ยนแปลง Steps I2–I11 (กระบวนการจัดประเภท frontmatter และ emit)
  • Bulk-HTML ingest mode or new CLI flagsโหมด bulk HTML ingest หรือ CLI flags ใหม่
  • Scaffold mode (--scaffold) changesการเปลี่ยนแปลงโหมด scaffold (--scaffold)

Acceptance Criteria — done whenเกณฑ์การยอมรับ — เสร็จเมื่อ

  • Given docs/feature/branch-guard-hook/plan.html, When /devstarter-vault-ingest is run on it, Then a sanitized vault note is emitted with title from HTML <h1>, body from document content only (nav/footer/script stripped), E4 applied, frontmatter gate shown, note written as .mdเมื่อรัน /devstarter-vault-ingest กับไฟล์ docs/feature/branch-guard-hook/plan.html จะได้ vault note ที่มีชื่อเรื่องจาก <h1> ของ HTML เนื้อหาจากส่วน body เท่านั้น (ลบ nav/footer/script ออก) ผ่านการ E4 sanitize และแสดง frontmatter gate ก่อนเขียนไฟล์ .md
  • Given an existing .md file, When /devstarter-vault-ingest is run on it, Then behaviour is unchanged (no regression)เมื่อรันกับไฟล์ .md ที่มีอยู่แล้ว พฤติกรรมจะไม่เปลี่ยนแปลง (ไม่มี regression)
  • A .pdf or .docx file still triggers the non-.md hard-stop errorไฟล์ .pdf หรือ .docx ยังคงแสดงข้อความ error เช่นเดิม
  • /devstarter-vault-ingest --scaffold behaviour unchangedพฤติกรรมของ /devstarter-vault-ingest --scaffold ไม่เปลี่ยนแปลง

Management Summaryสรุปสำหรับผู้บริหาร

📊 For management — business rationale, no technical detailสำหรับผู้บริหาร — เหตุผลเชิงธุรกิจ ไม่มีรายละเอียดทางเทคนิค

Why now — the business needทำไมต้องทำตอนนี้ — ความจำเป็นทางธุรกิจ

DevStarter generates 37 HTML documentation files across feature docs, ADRs, and release notes. These files contain valuable institutional knowledge that should be searchable from the Obsidian vault. The vault ingest tool currently rejects them entirely, creating a dead-end for knowledge capture from the primary documentation output of the system.DevStarter สร้างเอกสาร HTML 37 ไฟล์ครอบคลุม feature docs, ADR และ release notes ไฟล์เหล่านี้มีความรู้สำคัญที่ควรค้นหาได้จาก Obsidian vault แต่เครื่องมือ vault ingest ปัจจุบันปฏิเสธไฟล์เหล่านี้ทั้งหมด ทำให้ไม่สามารถจัดเก็บความรู้จากผลลัพธ์เอกสารหลักของระบบได้

Who benefitsใครได้ประโยชน์

All DevStarter users who want to search and recall project documentation from the Obsidian vault. The change makes the knowledge vault complete — both SDLC learnings (already working) and project documentation (currently blocked) can be captured and recalled.ผู้ใช้ DevStarter ทุกคนที่ต้องการค้นหาและเรียกดูเอกสารโครงการจาก Obsidian vault การเปลี่ยนแปลงนี้ทำให้ knowledge vault สมบูรณ์ — ทั้ง SDLC learnings (ที่ทำงานได้แล้ว) และเอกสารโครงการ (ที่ถูกบล็อกอยู่ขณะนี้) สามารถจัดเก็บและเรียกคืนได้

Cost of not doing thisผลกระทบหากไม่ดำเนินการ

The 37 existing HTML files and all future DevStarter-generated HTML docs remain inaccessible to vault search and recall. Users must manually convert files to Markdown before ingestion, which adds friction and discourages knowledge capture.ไฟล์ HTML 37 ไฟล์ที่มีอยู่และเอกสาร HTML ที่ DevStarter สร้างขึ้นในอนาคตทั้งหมดจะยังคงไม่สามารถค้นหาใน vault ได้ ผู้ใช้ต้องแปลงไฟล์เป็น Markdown ด้วยตนเองก่อนนำเข้า ซึ่งเพิ่มขั้นตอนและลดแรงจูงใจในการจัดเก็บความรู้
