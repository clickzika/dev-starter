Root Problem / Requirementต้นตอของปัญหา / ความต้องการ

Original Requirementความต้องการเดิม

Bring the Understand-Anything codebase-analysis tools (knowledge graph, dashboard, chat, change-impact, explain, onboard, domain, wiki knowledge) into DevStarter as native commands.นำเครื่องมือวิเคราะห์โค้ดเบสของ Understand-Anything (knowledge graph, dashboard, chat, การวิเคราะห์ผลกระทบ, explain, onboard, domain, wiki) เข้ามาใน DevStarter เป็นคำสั่งในตัว

Business Gapช่องว่างทางธุรกิจ

The tools existed only as a separate plugin with its own command set, forcing users to context-switch. DevStarter had no native entry point to codebase analysis.เครื่องมือมีอยู่เฉพาะในรูปปลั๊กอินแยกที่มีชุดคำสั่งของตัวเอง ทำให้ผู้ใช้ต้องสลับบริบท DevStarter ไม่มีจุดเข้าใช้การวิเคราะห์โค้ดเบสในตัว

How This Change Resolves Itการเปลี่ยนแปลงนี้แก้ไขอย่างไร

Eight thin /devstarter-understand* wrappers now delegate to the coexisting Understand-Anything plugin via the Skill tool, each with a preflight that detects the plugin and prompts to install it if absent. DevStarter stays pure Markdown — no vendoring, no build step.ตัวห่อหุ้มแบบบาง /devstarter-understand* จำนวนแปดตัวส่งต่อไปยังปลั๊กอิน Understand-Anything ที่อยู่ร่วมกันผ่าน Skill tool โดยแต่ละตัวมี preflight ตรวจหาปลั๊กอินและแจ้งเตือนให้ติดตั้งหากไม่มี DevStarter ยังเป็น Markdown ล้วน ไม่มีการ vendor ไม่มีขั้นตอน build

Implementation Notesบันทึกการพัฒนา

Key Decisions Madeการตัดสินใจสำคัญ

  • Path A (thin wrappers) chosen over full port — UA already coexists in ~/.claude/, so porting buys nothing. See ADR-0001.เลือกแนวทาง A (ตัวห่อหุ้มบาง) แทนการ port ทั้งหมด — UA อยู่ร่วมกันใน ~/.claude/ อยู่แล้ว การ port จึงไม่ได้ประโยชน์ ดู ADR-0001
  • Delegation uses the Skill tool with the plugin-namespaced skill name, not a typed slash command (the model cannot type slash commands).การส่งต่อใช้ Skill tool พร้อมชื่อ skill แบบ namespace ของปลั๊กอิน ไม่ใช่การพิมพ์ slash command (โมเดลพิมพ์ slash command ไม่ได้)
  • No naming exception needed — the devstarter- prefix keeps devstarter-understand-onboard distinct from devstarter-onboard.ไม่ต้องยกเว้นการตั้งชื่อ — prefix devstarter- ทำให้ devstarter-understand-onboard ต่างจาก devstarter-onboard

Technical Approachแนวทางเชิงเทคนิค

Each runbook runs a PHASE 1 preflight (check the plugin cache dir; if absent, show a one-step install prompt and stop) then a PHASE 2 delegate (invoke the namespaced UA skill via the Skill tool, forwarding the user's arguments). No DevStarter logic re-implements analysis.แต่ละ runbook รัน preflight ใน PHASE 1 (ตรวจ cache dir ของปลั๊กอิน ถ้าไม่มีให้แสดงคำแนะนำติดตั้งขั้นตอนเดียวแล้วหยุด) จากนั้น PHASE 2 ส่งต่อ (เรียก skill UA แบบ namespace ผ่าน Skill tool พร้อมส่ง argument ของผู้ใช้) ไม่มีตรรกะของ DevStarter ที่เขียนการวิเคราะห์ใหม่

Deviations from Planความแตกต่างจากแผน

One addition beyond the plan: after an advisor review, the delegation step was tightened to name the Skill-tool mechanism and namespaced skill explicitly (initially it read like a typed slash command). Committed as a follow-up fix.มีหนึ่งสิ่งที่เพิ่มจากแผน: หลังการรีวิวจากที่ปรึกษา ขั้นตอนการส่งต่อถูกปรับให้ระบุกลไก Skill tool และ skill แบบ namespace อย่างชัดเจน (ตอนแรกอ่านเหมือน slash command ที่พิมพ์) commit เป็นการแก้ตามมา
