Problem / Requirementปัญหา / ความต้องการ

Root Problemต้นตอของปัญหา

The Understand-Anything codebase-analysis tools (knowledge graph, dashboard, chat, change-impact, onboarding, domain extraction, wiki analysis) are valuable for understanding unfamiliar projects, but they live as a separate plugin with their own command set. DevStarter users have no native entry point to them and must context-switch to a second toolset.เครื่องมือวิเคราะห์โค้ดเบสของ Understand-Anything (knowledge graph, dashboard, chat, การวิเคราะห์ผลกระทบ, onboarding, การสกัด domain, การวิเคราะห์ wiki) มีคุณค่าต่อการทำความเข้าใจโปรเจกต์ที่ไม่คุ้นเคย แต่อยู่ในรูปปลั๊กอินแยกที่มีชุดคำสั่งของตัวเอง ผู้ใช้ DevStarter ไม่มีจุดเข้าใช้งานในตัวและต้องสลับไปใช้ชุดเครื่องมือที่สอง

Business Requirementความต้องการทางธุรกิจ

Expose all 8 Understand-Anything commands as native DevStarter commands so users get codebase analysis from inside their existing workflow, without copying or rebuilding the underlying tool.นำคำสั่ง Understand-Anything ทั้ง 8 ตัวมาเปิดใช้เป็นคำสั่ง DevStarter ในตัว เพื่อให้ผู้ใช้ได้การวิเคราะห์โค้ดเบสจากภายใน workflow เดิม โดยไม่ต้องคัดลอกหรือ build เครื่องมือเบื้องหลังใหม่

User Storyเรื่องราวของผู้ใช้

As a DevStarter user joining an unfamiliar codebaseผู้ใช้ DevStarter ที่เพิ่งเข้ามาดูโค้ดเบสที่ไม่คุ้นเคย,
I want to analyse, visualise, and ask questions about the project using DevStarter commandsวิเคราะห์ มองเห็นภาพ และถามคำถามเกี่ยวกับโปรเจกต์ด้วยคำสั่ง DevStarter,
So that I understand the system fast without leaving my established workflowฉันเข้าใจระบบได้เร็วโดยไม่ต้องออกจาก workflow ที่ใช้อยู่.

Acceptance Criteriaเกณฑ์การยอมรับ

  • Given the plugin is installed, When any /devstarter-understand* command runs, Then the matching Understand-Anything command executes with arguments passed through.เมื่อปลั๊กอินถูกติดตั้งแล้ว และเรียกคำสั่ง /devstarter-understand* ใด ๆ ระบบจะเรียกคำสั่ง Understand-Anything ที่ตรงกันพร้อมส่ง argument ต่อไป
  • Given the plugin is not installed, When a /devstarter-understand* command runs, Then a clear one-step install prompt appears first.เมื่อยังไม่ได้ติดตั้งปลั๊กอิน และเรียกคำสั่ง /devstarter-understand* ระบบจะแสดงคำแนะนำติดตั้งแบบขั้นตอนเดียวที่ชัดเจนก่อน
  • Given the build is complete, When devstarter-menu.md is opened, Then all 8 commands appear as entries.เมื่อสร้างเสร็จแล้ว และเปิด devstarter-menu.md คำสั่งทั้ง 8 ตัวจะปรากฏเป็นรายการ
  • Given DevStarter conventions, When files are reviewed, Then every command keeps the devstarter- prefix and the one-command-two-files rule.ตามแบบแผนของ DevStarter เมื่อรีวิวไฟล์ คำสั่งทุกตัวจะคงคำนำหน้า devstarter- และกฎหนึ่งคำสั่งสองไฟล์

Proposed Solutionแนวทางแก้ไขที่เสนอ

Approachแนวทาง

Path A — thin wrappers. For each of the 8 commands, create a SKILL.md entry point and a matching SDLC runbook. Each runbook runs a preflight that checks whether the Understand-Anything plugin is installed (looks for its cache directory); if missing, it shows a one-step install prompt and stops; if present, it forwards the user's arguments to the underlying command. Register all 8 in devstarter-menu.md and add an install hint to install.sh. No Understand-Anything source is copied.แนวทาง A — ตัวห่อหุ้มแบบบาง สำหรับคำสั่งทั้ง 8 ตัว สร้างจุดเข้าใช้ SKILL.md และ SDLC runbook ที่ตรงกัน แต่ละ runbook จะรัน preflight เพื่อตรวจว่าติดตั้งปลั๊กอิน Understand-Anything หรือยัง (มองหาไดเรกทอรี cache) ถ้าไม่มี จะแสดงคำแนะนำติดตั้งขั้นตอนเดียวแล้วหยุด ถ้ามี จะส่ง argument ของผู้ใช้ต่อไปยังคำสั่งเบื้องหลัง จากนั้นลงทะเบียนทั้ง 8 ตัวใน devstarter-menu.md และเพิ่มคำแนะนำติดตั้งใน install.sh ไม่มีการคัดลอกซอร์สของ Understand-Anything

Why This Approachเหตุผลที่เลือกแนวทางนี้

Understand-Anything already installs into the same ~/.claude/ home as DevStarter, so the two coexist with zero porting. Thin wrappers deliver the full capability set while keeping DevStarter pure Markdown — no build step, no vendored TypeScript to maintain, and upstream updates flow through automatically.Understand-Anything ติดตั้งลงใน home ~/.claude/ เดียวกับ DevStarter อยู่แล้ว ทั้งสองจึงอยู่ร่วมกันได้โดยไม่ต้อง port ตัวห่อหุ้มแบบบางส่งมอบความสามารถครบชุดพร้อมคง DevStarter ให้เป็น Markdown ล้วน ไม่มีขั้นตอน build ไม่มีโค้ด TypeScript ที่ต้องดูแล และอัปเดตจากต้นทางไหลผ่านมาอัตโนมัติ

Alternatives Consideredทางเลือกอื่นที่พิจารณา

OptionReason Rejected
Full port / vendor the TS monorepoPort / นำซอร์ส TS monorepo ทั้งหมดเข้ามาBreaks DevStarter's no-build Markdown model, adds Node/pnpm bootstrap, creates vendored drift and naming-convention churn, and carries MIT attribution overhead — all for capability the coexisting plugin already provides.ทำลายโมเดล Markdown แบบไม่ build ของ DevStarter เพิ่ม bootstrap Node/pnpm สร้างการ drift จากการ vendor และความวุ่นวายเรื่องการตั้งชื่อ และมีภาระเรื่อง attribution ของ MIT ทั้งหมดนี้เพื่อความสามารถที่ปลั๊กอินซึ่งอยู่ร่วมกันมีให้อยู่แล้ว
Native Markdown agents (reimplement)สร้าง agent เป็น Markdown ใหม่ (เขียนซ้ำ)Loses deterministic tree-sitter parsing and the React dashboard — the core value — and reinvents a maintained tool, worse.สูญเสียการ parse แบบ deterministic ด้วย tree-sitter และ dashboard React ซึ่งเป็นคุณค่าหลัก และเป็นการประดิษฐ์เครื่องมือที่มีคนดูแลอยู่ใหม่แบบด้อยกว่า

Implementation Stepsขั้นตอนการพัฒนา

  1. Create docs/adr/0001-understand-anything-integration.html recording the Path A decision.สร้าง docs/adr/0001-understand-anything-integration.html บันทึกการตัดสินใจแนวทาง A
  2. Create the 8 skills/devstarter-understand*/SKILL.md entry points following the existing thin-skill pattern.สร้างจุดเข้าใช้ skills/devstarter-understand*/SKILL.md ทั้ง 8 ตัวตามรูปแบบ thin-skill เดิม
  3. Create the 8 sdlc/devstarter-understand*.md runbooks with UA preflight check and argument pass-through.สร้าง runbook sdlc/devstarter-understand*.md ทั้ง 8 ตัว พร้อม preflight ตรวจ UA และการส่ง argument ต่อ
  4. Add 8 rows to devstarter-menu.md.เพิ่ม 8 แถวใน devstarter-menu.md
  5. Add a plugin install hint to install.sh.เพิ่มคำแนะนำติดตั้งปลั๊กอินใน install.sh
  6. Update CLAUDE.md command list and Recently Shipped table (v5.6.0).อัปเดตรายการคำสั่งและตาราง Recently Shipped ใน CLAUDE.md (v5.6.0)

Dependenciesสิ่งที่ต้องพึ่งพา

  • Understand-Anything plugin (external, MIT) — runtime dependency, detected at command timeปลั๊กอิน Understand-Anything (ภายนอก, MIT) — dependency ตอนรันไทม์ ตรวจขณะเรียกคำสั่ง
  • Node.js — only for the dashboard command, required by the plugin not DevStarterNode.js — เฉพาะคำสั่ง dashboard ปลั๊กอินต้องการ ไม่ใช่ DevStarter

Testing Approachแนวทางการทดสอบ

Manual verification: run each /devstarter-understand* command with the plugin absent (expect install prompt) and present (expect pass-through). Confirm the 8 menu rows render and that existing /devstarter-onboard is unaffected.ตรวจด้วยมือ: เรียกแต่ละคำสั่ง /devstarter-understand* ตอนไม่มีปลั๊กอิน (คาดว่าได้คำแนะนำติดตั้ง) และตอนมี (คาดว่าส่งต่อ) ยืนยันว่ารายการเมนู 8 แถวแสดงผลและ /devstarter-onboard เดิมไม่ได้รับผลกระทบ
