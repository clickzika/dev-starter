Notion TaskBUG: stale commands/ references from v3.0.0 migration

Summary

The v3.0.0 release migrated all 41 slash commands from the flat commands/devstarter-*.md format to the Claude Code official skills directory format (skills/devstarter-*/SKILL.md). The migration removed commands/ from the git repository but left the local directory as an untracked artifact. Additionally, four files were not updated to reflect the new path, causing stale references that would mislead users and break tooling on a clean clone.

Root Cause

The v3.0.0 migration did not include a cleanup step for the local commands/ folder, and four files retained hardcoded references to the old path:

  • scripts/dev-setup.sh:39 — validated commands/ existence; would error on any clean clone after v3.0.0
  • sdlc/devstarter-doctor.md:50 — checked ~/.claude/commands/devstarter-*.md (wrong path, wrong count of 25)
  • skills/devstarter-export/SKILL.md:54,147 — verification loop + display text referenced commands/
  • skills/devstarter-import/SKILL.md:91 — counted ~/.claude/commands/*.md (wrong path, expected wrong count)

Fix Applied

  • Deleted local commands/ folder (41 untracked .md files — all content exists in skills/)
  • scripts/dev-setup.sh — now validates skills/ not commands/; error message updated
  • sdlc/devstarter-doctor.md — section 3 title updated to "Skills (41 required)"; path updated to ~/.claude/skills/devstarter-*/SKILL.md
  • skills/devstarter-export/SKILL.md — verification loop now checks ~/.claude/skills/devstarter-$f/SKILL.md; display text updated to reflect skills/ with 41 entries
  • skills/devstarter-import/SKILL.md — count now reads ~/.claude/skills/*/SKILL.md; expected count updated to 41
