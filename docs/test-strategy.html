Scope & Objectives

DevStarter is a Markdown + Bash toolkit with no compiled code. Traditional unit and integration test frameworks do not apply. Testing is primarily manual workflow validation — running commands end-to-end and verifying output correctness, completeness, and cross-reference integrity.

Current State

No automated test suite exists (see OI-001 in BRD). All validation is manual pre-release. Future goal: a bash-based validation script that checks cross-references, required fields, and file naming conventions.

Test Objectives

  • Verify all 24 slash commands route to the correct SDLC runbook
  • Verify all agent files read correctly by Claude Code
  • Verify install.sh runs successfully on all 3 platforms
  • Verify checkpoints save and resume correctly
  • Verify gate approval stops are enforced — no command skips a gate
  • Verify all document files use the standard template
  • Verify all internal cross-references use the devstarter- prefix

Manual Test Procedures

TP-001: Smoke Test

  1. Clone fresh from release remote to a temp directory
  2. Run bash install.sh
  3. Open Claude Code in any project folder
  4. Type: /devstarter-menu
  5. Pass: Menu displays all 20 options correctly
  6. Fail: Error, missing options, or wrong routing

TP-002: Checkpoint Resume Test

  1. Start /devstarter-new React test app
  2. Let it reach Gate 1 approval and approve
  3. Let it start Gate 2 document generation
  4. Close the Claude Code session mid-generation
  5. Reopen Claude Code in the same directory
  6. Type: continue
  7. Pass: Agent reads progress.json, announces resume, continues from last task without repeating approved work
  8. Fail: Agent restarts from beginning or fails to detect checkpoint

TP-003: Gate Enforcement Test

  1. Start /devstarter-new and reach Gate 1
  2. Do NOT type "approve" — instead type "continue" or any other phrase
  3. Pass: Agent holds at the gate, asks for explicit "approve" or "revise"
  4. Fail: Agent proceeds past the gate without approval

TP-004: Cross-Reference Integrity

Future: Automated Testing

Planned validation script (addresses OI-001):

  • validate-commands.sh — check every entry in devstarter-menu.md has a matching sdlc/ runbook
  • validate-prefixes.sh — grep for un-prefixed agent/command references
  • validate-templates.sh — verify all docs/ HTML files contain the CSS variables block
  • validate-json.sh — verify progress.json is valid JSON with required fields

These would run as a GitHub Actions workflow on every PR to the dev repo origin.
