Overview
What this release adds and why
v2.2.0 adds structured requirement intake to all four core workflows — New Project, Add Feature, Modify Feature, and Fix Bug. Instead of answering ad-hoc questions one by one, you can now write your requirements in a structured .md template file beforehand and pass it directly to the command. Claude reads the file, skips all interactive intake questions, and goes straight to impact analysis or setup.
🌟 Key Benefits
  • 📋
    Pre-written requirements Write your PRD or bug report in a template file, then pass it to the command — no interactive Q&A.
  • 🔍
    Auto-detection of change type Claude reads the file content and auto-detects whether it's a new feature, modify, or bug fix from keywords.
  • ⏬
    Answer carry-forward After intake approval, all overlapping SDLC questions are skipped automatically — no duplicate input.
  • 🔨
    Interactive fallback If no file is passed, Claude still walks you through each section interactively and saves the result to memory/.

⚡ File-Arg Pattern MODIFIED

Both /devstarter-new and /devstarter-change now check for a file argument before any other logic. If you pass a .md or .txt file path, Claude reads it, extracts all requirements, and shows a pre-filled INTAKE SUMMARY for approval — skipping the entire interactive intake flow.

Type auto-detection from file content:

Keywords in fileDetected type
AS-IS, TO-BE, modify, change existingModify Feature
bug, error, fix, brokenFix Bug
(everything else)Add Feature
🔌 SECTION 0 / A-SECTION 0 / C-SECTION 0 MODIFIED
New intake sections prepended to three SDLC runbooks. Priority order in each:
  1. File arg — read file → show pre-filled INTAKE SUMMARY → wait for approval → skip to setup/analysis
  2. Inline description — extract from MODE 3 text → pre-fill → confirm
  3. Interactive — read matching template → section by section → save to memory/ → show summary

How to Update
From v2.1.x to v2.2.0
Run /devstarter-update
In any Claude Code session, run:
/devstarter-update
This pulls the latest version and installs all new files to ~/.claude/. The agents/custom/ folder is preserved during updates.
No breaking changes
v2.2.0 is fully backward compatible. Existing workflows are unchanged — the new SECTION 0 blocks activate only when a file arg is detected. All existing commands, agents, and config files continue to work without modification.

Usage Examples
How to use the new intake pattern
New Project with PRD file
# 1. Copy the template to your project folder cp ~/.claude/templates/intake/devstarter-intake-new-project.md prd.md # 2. Fill in your answers in prd.md # 3. Start the project (Claude reads the file automatically) /devstarter-new prd.md
Add Feature with requirements file
# 1. Copy the template cp ~/.claude/templates/intake/devstarter-intake-add-feature.md dark-mode.md # 2. Fill in the sections # 3. Pass to devstarter-change /devstarter-change dark-mode.md
Report a bug
# 1. Copy the bug template cp ~/.claude/templates/intake/devstarter-intake-fix-bug.md login-bug.md # 2. Fill in steps to reproduce, expected vs actual, severity # 3. Claude auto-detects it's a bug from file content /devstarter-change login-bug.md

Known Issues

  • 🔶
    No "skip intake" mechanism No documented way to bypass intake entirely in a session. Planned for v2.2.1.
