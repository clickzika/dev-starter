## Key Facts for Agents

### This is NOT a software application
When agents work on this project, they are editing:
- Markdown workflow files (`.md`)
- Bash scripts (`.sh`)
- HTML templates (`.html`)
- JSON config files (`.json`)

There is no compilation, no build step, no test runner, no deployment pipeline in the traditional sense.

### "Releasing" means publishing to users
Release = bump VERSION + update CHANGELOG + commit + push main + git tag.
The `publish.sh` script and `/devstarter-release` command handle this.

### Naming convention вЂ” MANDATORY
All files MUST be prefixed with `devstarter-`:
- Agents: `devstarter-[name].md`
- Commands: `devstarter-[name].md`
- SDLC runbooks: `devstarter-[name].md`

Cross-references MUST use the full prefixed name.
Do NOT create files without the `devstarter-` prefix (except root files: CLAUDE.md, README.md, CHANGELOG.md, VERSION, USER.md).

### Slash commands vs SDLC runbooks
- `commands/devstarter-X.md` вЂ” the slash command file (thin, routes to SDLC)
- `sdlc/devstarter-X.md` вЂ” the actual workflow runbook (thick, full logic)
- When adding a new command: create BOTH files. Register in `devstarter-menu.md`.
